This changes everything. Instead of designing a whole compiler, we are designing a **drop-in replacement for the `glyim-typeck` crate** that perfectly conforms to the existing pipeline's API, while internally leveraging the V2/V3 metacompiler architecture (CHR, Optics, Zippering, `glyim-query`, `glyim-macro-core`).

The brilliance of this design is that **the pipeline doesn't need to know about CHR, Optics, or Staging**. It only cares about `Vec<HirType>` and `HashMap<ExprId, Vec<HirType>>`. All the academic innovations happen *inside* the typechecker, manifesting externally as better error messages, correct generic resolution, and optional `.rodata` metadata for the JIT/Codegen.

Here is the complete, pipeline-compliant architectural specification for **Glyim Typeck V3**.

# THE GLYIM TYPECK V3 SPECIFICATION: THE PIPELINE-COMPLIANT METACOMPILER

## 1. The Public API (`lib.rs`)
*Drop-in replacement. Zero changes to `pipeline.rs` required (unless they want the reflection output).*

```rust
use glyim_hir::{Hir, HirType, types::ExprId};
use glyim_interner::Interner;
use std::collections::HashMap;

/// The output expected by `glyim-compiler::pipeline`.
pub struct TypeCheckOutput {
    /// Maps every ExprId to its inferred HirType.
    pub expr_types: Vec<HirType>,
    /// Maps generic call/method call ExprIds to their concrete type arguments.
    pub call_type_args: HashMap<ExprId, Vec<HirType>>,
    
    /// V3 Addition: Reflection metadata for the codegen/JIT.
    /// If the pipeline is configured for runtime reflection, it reads this.
    /// Otherwise, it's ignored.
    pub reflect_metadata: Vec<crate::reflect::TypeMetaSoA>,
    
    /// V3 Addition: Items generated by comptime blocks.
    /// The pipeline must append these to the HIR before monomorphization.
    pub generated_items: Vec<glyim_hir::HirItem>,
}

pub struct TypeChecker {
    interner: Interner,
}

impl TypeChecker {
    pub fn new(interner: Interner) -> Self {
        Self { interner }
    }

    /// The exact API the pipeline expects.
    pub fn check(
        &mut self,
        hir: &Hir,
    ) -> Result<TypeCheckOutput, Vec<crate::diagnostics::TypeError>> {
        // Initialize the demand-driven database
        let mut db = crate::db::TyDatabase::new(self.interner.clone());
        
        // Run the query-driven type checking
        db.check_module(hir)
    }
}
```

## 2. The Internal Engine (`db.rs`)
*Orchestrates V2 Elaboration + V3 Metaprogramming, outputs standard `HirType`s.*

The pipeline assumes a single pass, but our typechecker is demand-driven. We wrap the entire check into a single top-level query execution.

```rust
use crate::ty::{Ty, TyKind, TyArena};
use crate::unify::UnificationTable;
use crate::chr::{ChrStore, Goal};
use crate::elab::ElabContext;
use crate::freeze;
use crate::reflect;
use crate::diagnostics::TypeError;
use glyim_hir::{Hir, HirType, HirItem, types::ExprId};
use glyim_interner::Interner;
use std::collections::HashMap;

pub struct TyDatabase {
    pub arena: TyArena,
    pub interner: Interner,
}

impl TyDatabase {
    pub fn new(interner: Interner) -> Self {
        Self {
            arena: TyArena::new(),
            interner,
        }
    }

    pub fn check_module(&mut self, hir: &Hir) -> Result<super::TypeCheckOutput, Vec<TypeError>> {
        // 1. Registration Phase
        let mut chr_store = ChrStore::new();
        for item in &hir.items {
            self.register_item(item, &mut chr_store);
        }

        // 2. Elaboration Phase
        let mut unification = UnificationTable::new();
        let mut elab = ElabContext::new(
            self,
            hir,
            &mut unification,
            &mut chr_store,
        );

        // Elaborate all items (may trigger comptime/macro evaluation)
        for item in &hir.items {
            elab.elaborate_item(item);
        }

        // 3. Solving Phase
        if let Err(_) = chr_store.solve(&self.arena) {
            // Collect unsolved goals as errors
            elab.collect_goal_errors();
        }

        // 4. Freeze Phase (Translate internal Ty -> Pipeline's HirType)
        let expr_types = freeze::resolve_expr_types(&self.arena, &unification, &elab.expr_types_map);
        let call_type_args = freeze::resolve_call_args(&self.arena, &unification, &elab.call_type_args_map);

        // 5. Reflection Metadata Generation
        let reflect_metadata = reflect::generate_metadata(&chr_store.proven_goals, &self.arena);

        if elab.errors.is_empty() {
            Ok(super::TypeCheckOutput {
                expr_types,
                call_type_args,
                reflect_metadata,
                generated_items: elab.generated_items,
            })
        } else {
            Err(elab.errors)
        }
    }

    fn register_item(&mut self, item: &HirItem, chr_store: &mut ChrStore) {
        match item {
            HirItem::Struct(def) => {
                let ty = self.arena.alloc(TyKind::Named(def.name));
                // Register CHR rules for @reflectable, etc.
            }
            HirItem::Impl(def) => {
                // Generate CHR rules for trait implementations
                chr_store.add_rules(self.lower_impl_to_chr_rules(def));
            }
            _ => {}
        }
    }
}
```

## 4. The CHR Solver (`chr.rs`)
*Proves trait goals, state transitions, and reflection queries.*

```rust
use crate::ty::{Ty, TyArena};
use glyim_interner::Symbol;
use std::collections::HashSet;

#[derive(Clone, Debug, PartialEq, Eq, Hash)]
pub enum Goal {
    TraitImpl(Symbol, Vec<Ty>),
    StateTransition(Symbol, Ty, Ty),
    
    // V3 Reflection Goals
    Reflectable(Ty),
    HasField(Ty, Symbol),      // Prove type T has field F
}

pub struct ChrStore {
    rules: Vec<ChrRule>,
    pub pending_goals: Vec<Goal>,
    pub proven_goals: HashSet<Goal>,
}

impl ChrStore {
    pub fn solve(&mut self, arena: &TyArena) -> Result<(), ()> {
        while let Some(goal) = self.pending_goals.pop() {
            if self.proven_goals.contains(&goal) { continue; }
            
            let mut rule_matched = false;
            for rule in &self.rules {
                if rule.matches(&goal) {
                    rule_matched = true;
                    self.proven_goals.insert(goal);
                    break;
                }
            }
            
            if !rule_matched {
                return Err(()); // Type error!
            }
        }
        Ok(())
    }
}
```

## 5. The Elaborator (`elab/mod.rs`)
*Walks HIR, emits Goals, outputs raw `Ty`s indexed by `ExprId`.*

The pipeline requires types for *every* `ExprId`. We maintain internal maps, which the Freeze phase later converts to `Vec<HirType>`.

```rust
use crate::ty::{Ty, TyKind, TyArena};
use crate::unify::UnificationTable;
use crate::chr::{ChrStore, Goal};
use crate::db::TyDatabase;
use crate::diagnostics::TypeError;
use glyim_hir::{Hir, HirExpr, HirItem, types::ExprId};
use glyim_diag::Span;
use std::collections::HashMap;

pub struct ElabContext<'a> {
    db: &'a mut TyDatabase,
    hir: &'a Hir,
    unification: &'a mut UnificationTable,
    chr_store: &'a mut ChrStore,
    
    // Internal maps (keyed by ExprId)
    pub expr_types_map: HashMap<ExprId, Ty>,
    pub call_type_args_map: HashMap<ExprId, Vec<Ty>>,
    
    pub errors: Vec<TypeError>,
    pub generated_items: Vec<glyim_hir::HirItem>,
}

impl<'a> ElabContext<'a> {
    pub fn elaborate_item(&mut self, item: &HirItem) {
        if let HirItem::Fn(def) = item {
            self.synth_expr(&def.body);
        }
    }

    pub fn synth_expr(&mut self, expr: &HirExpr) -> Ty {
        let ty = match expr {
            HirExpr::IntLit { .. } => self.db.arena.alloc(TyKind::Int),
            
            HirExpr::MethodCall { receiver, method_name, args, id, span } => {
                let receiver_ty = self.synth_expr(receiver);
                let ret_ty = self.unification.new_var(&mut self.db.arena, *span);
                
                // Emit goal for method resolution
                self.chr_store.pending_goals.push(Goal::TraitImpl(*method_name, vec![receiver_ty, ret_ty]));
                
                // Record call type args for the desugarer/monomorphizer
                self.call_type_args_map.insert(*id, vec![receiver_ty]);
                
                ret_ty
            }

            // ─── V3: Reflection Support ───
            HirExpr::Call { callee, args, id, span } => {
                // If this is a call to `reflect_get(obj, "age")`
                if self.is_reflect_get(callee) {
                    self.elaborate_reflect_get(args, *span, *id)
                } else {
                    self.elaborate_normal_call(callee, args, *id, *span)
                }
            }

            _ => self.db.arena.alloc(TyKind::Error),
        };

        // CRITICAL: Ensure every ExprId gets a type
        if let Some(id) = expr.expr_id() {
            self.expr_types_map.insert(id, ty);
        }
        
        ty
    }

    fn elaborate_reflect_get(&mut self, args: &[HirExpr], span: Span, call_id: ExprId) -> Ty {
        // args[0] = receiver, args[1] = field name (string literal)
        let receiver_ty = self.synth_expr(&args[0]);
        let field_name = self.extract_string_lit(&args[1]);
        let result_ty = self.unification.new_var(&mut self.db.arena, span);

        // Emit a HasField goal! The CHR solver will prove this.
        self.chr_store.pending_goals.push(Goal::HasField(receiver_ty, field_name));

        // If the receiver type is concretely known, we can immediately 
        // unify the result type, making it transparent to the monomorphizer!
        if let TyKind::App(sym, _) = self.db.arena.get(receiver_ty) {
            if let Some(field_ty) = self.lookup_field_type(*sym, field_name) {
                self.unification.unify(&mut self.db.arena, result_ty, field_ty, span, &mut |_| {}).ok();
            }
        }
        
        // The desugarer/monomorphizer just sees the resolved type (e.g., Int)
        result_ty
    }
}
```

## 6. The Freeze Phase (`freeze.rs`)
*Translates internal `Ty` to the Pipeline's `HirType`.*

The pipeline demands `Vec<HirType>` indexed by `ExprId::as_usize()`. We must map our internal types back, resolving all inference variables.

```rust
use crate::ty::{Ty, TyKind, TyArena};
use crate::unify::UnificationTable;
use glyim_hir::{HirType, types::ExprId};
use glyim_interner::Interner;
use std::collections::HashMap;

pub fn resolve_expr_types(
    arena: &TyArena,
    unification: &UnificationTable,
    elab_map: &HashMap<ExprId, Ty>,
) -> Vec<HirType> {
    let mut expr_types = Vec::new();
    
    // Find max ExprId to pre-allocate the vector (Pipeline requirement)
    let max_id = elab_map.keys().map(|id| id.as_usize()).max().unwrap_or(0);
    expr_types.resize(max_id + 1, HirType::Error);

    for (id, ty) in elab_map {
        let resolved_ty = unification.find(arena, *ty);
        expr_types[id.as_usize()] = resolve_ty(arena, unification, resolved_ty);
    }
    
    expr_types
}

fn resolve_ty(arena: &TyArena, uni: &UnificationTable, ty: Ty) -> HirType {
    let ty = uni.find(arena, ty); // Path compression
    
    match arena.get(ty) {
        TyKind::Int => HirType::Int,
        TyKind::Bool => HirType::Bool,
        TyKind::Named(sym) => HirType::Named(*sym),
        TyKind::App(sym, args) => HirType::Generic(
            *sym,
            args.iter().map(|a| resolve_ty(arena, uni, *a)).collect(),
        ),
        TyKind::Fn(params, ret) => HirType::Func(
            params.iter().map(|p| resolve_ty(arena, uni, *p)).collect(),
            Box::new(resolve_ty(arena, uni, *ret)),
        ),
        TyKind::Infer => HirType::Error, // Unresolved hole
        _ => HirType::Error,
    }
}

pub fn resolve_call_args(
    arena: &TyArena,
    unification: &UnificationTable,
    elab_map: &HashMap<ExprId, Vec<Ty>>,
) -> HashMap<ExprId, Vec<HirType>> {
    elab_map.iter().map(|(id, tys)| {
        let hir_tys = tys.iter().map(|t| resolve_ty(arena, unification, *t)).collect();
        (*id, hir_tys)
    }).collect()
}
```

## 9. How V3 Features Map to the Pipeline

| V3 Feature | How it works inside `glyim-typeck` | What the Pipeline Sees |
|---|---|---|
| **CHR Solving** | Proves `TraitImpl` and `StateTransition` goals. | Correct `expr_types` for method calls. |
| **Reflective `getField`** | Emits `HasField` goal, unifies result `Ty`. | Standard `HirType::Int` (or whatever the field type is). |
| **Optics / Lenses** | Used internally for comptime generation. | Not seen; pure compile-time construct. |
| **Comptime Codegen** | Evaluates blocks, generates `HirItem`s. | `TypeCheckOutput::generated_items` appended to HIR before mono. |
| **SoA Metadata** | Generated from `proven_goals`. | `TypeCheckOutput::reflect_metadata` passed to codegen/JIT. |
| **Zippering Errors** | Used when `unify` fails. | Beautiful `TypeError` with structural diffs. |

This design achieves the ultimate goal: **Glyim's pipeline remains completely stable, while the typechecker becomes a state-of-the-art metacompiler.** The monomorphizer and desugarer operate exactly as before, consuming the `Vec<HirType>` and `HashMap<ExprId, Vec<HirType>>`, oblivious to the fact that those types were derived via CHR logic programming and profunctor optics.
