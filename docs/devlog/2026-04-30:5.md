# 2026-05-01: v0.5.0 Language Foundation – Monomorphization, Method Calls, While, Casts, &self, Generic Types

## What I Did

- **Monomorphization Pipeline** — Completed the monomorphization pass for generic functions and structs. Fixed five critical bugs:
  - **Queue seeding**: the work queue now scans function bodies for generic calls instead of relying solely on `call_type_args` from the type checker.
  - **Structural type-param binding**: inference now walks parameter types to match type variables rather than assuming positional correspondence.
  - **Key collision in `fn_mangle_map`**: changed the map to key on `(Symbol, Vec<HirType>)` instead of just `Symbol`, preventing two specializations of the same function from overwriting each other.
  - **Nested generic calls**: after `specialize_fn` substitutes types in the body, the work‑queue scanner can now discover inner generic calls.
  - **Duplicate queue entries**: added a `fn_queued` set to deduplicate before pushing.

- **Codegen Architecture** — Refactored `Codegen::generate` and `generate_for_tests` into a strict three‑pass pipeline: (1) register struct/enum/extern types, (2) forward‑declare all functions via `declare_fn`, (3) emit function bodies. This guarantees that monomorphized functions are visible to callers regardless of their position in the HIR item list. Removed a duplicate `add_function` call in `codegen_fn` that was creating conflicting function declarations.

- **While Loops** — Added full support for `while { condition } { body }`:
  - Lexer: `"while"` → `KwWhile`
  - Syntax: `WhileExpr` node type
  - Parser: `parse_while` in `complex.rs`
  - HIR: `HirExpr::While` variant with lowering
  - Codegen: three‑block pattern (`while.cond`, `while.body`, `while.end`) leveraging the existing alloca‑based mutable variables, no φ‑nodes required.

- **`as` Cast Codegen** — Replaced the stub that returned constant zero with proper LLVM cast instructions:
  - `Int` → `Float`: `sitofp`
  - `Float` → `Int`: `fptosi`
  - `Int`/`Ptr` conversions: `inttoptr`, `ptrtoint`
  - Fixed the parser mapping for all `TypeExpr` variants (was missing `Int`, `Float`, `Bool`, `Str`, `Unit`, `RawPtr`) so that `42 as f64` correctly resolves to `Float`.

- **Method Call Desugaring** — The parser and HIR already recognized `obj.method(args)`. Lowering now correctly maps it to `HirExpr::MethodCall`. Codegen constructs a mangled function name (`TypeName_methodName`) from the receiver type and emits the call. Type checker now iterates `HirItem::Impl` and pushes each method into `fns` so they are visible during semantic analysis.

- **`&self` / `&mut self` Parameters** — In impl blocks, `&self` and `&mut self` are now parsed and desugared to a normal parameter with the impl target type. Added `Amp` token to the lexer (single‑char `&`) and `SyntaxKind::Amp`. The impl target type is propagated through `parse_fn_def_with_attrs` so that `&self` receives the correct type annotation.

- **Comparison Operators Return `Bool`** — The type checker now distinguishes comparison operators (`==`, `!=`, `<`, `>`, `<=`, `>=`) from arithmetic operators, returning `HirType::Bool` instead of `Int`. This fixes the `e2e_while_loop` test where `i < 5` was incorrectly typed as `Int`.

- **Generic Struct Support** — The parser already handled `struct Vec<T> { … }` and `impl<T> Vec<T>`. Fixed lowering to preserve `type_params` for structs and enums, and to use actual type annotations for fields instead of hardcoded `HirType::Int`. The type checker now substitutes generic field types when inferring struct literal types.

- **Field Assignment (`self.field = value`)** — Added the `StmtKind::AssignField` AST node, `HirStmt::AssignField` HIR node, and full pipeline support (parsing, lowering, type checking, monomorphisation, codegen). The block parser in `complex.rs` now recognizes `ExprKind::FieldAccess` as a valid assignment target. Codegen for `AssignField` finds the struct type, computes the field pointer via GEP, and stores the value.

- **Heap‑Allocated Struct Literals** — Changed `codegen_struct_lit` from stack‑allocation (`alloca`) to heap‑allocation via `glyim_alloc` (or `malloc`). This ensures structs returned from functions remain valid after the call returns. The size is computed from the LLVM struct type.

- **String Type** — Implemented `String` as a struct wrapping `Vec<u8>` with `new()` and `len()` methods. The parser correctly handles `Vec<u8>` as a field type now (generic type arguments on struct fields were previously unsupported).

- **Generic Wrapper Tests** — Added integration tests for `Wrapper<T>` with `bool` and `i64` instantiations, proving that monomorphization works end‑to‑end with method calls, field access, and `if` conditions.

- **Cleaned Debug Output** — Removed `eprintln!` and `println!` debug statements from monomorphize, codegen, type checker, and pipeline files. Fixed `SyntaxKind::COUNT` mismatches caused by adding new token variants.

## Problems Hit

- **Borrow checker conflicts in monomorphize** — The `MonoContext` had multiple borrow conflicts when building the mangle maps and iterating over `fn_specs`/`struct_specs`. Required pre‑computing lookups and cloning data before mutable operations.

- **`HirType` missing `Hash` and `Eq`** — Using `Vec<HirType>` as a map key required those derives, which were missing. Added them to the derive macro.

- **Three‑pass codegen replacement repeatedly failed** — The Python replacement code didn't match the exact text in `mod.rs`, requiring multiple attempts to apply the correct pattern.

- **`&` token lexed as `Error`** — The lexer's single‑character token match was missing `'&'`, causing `&self` to be parsed as an error. Added `'&' => SyntaxKind::Amp`.

- **Parser not recognizing field‑access assignment targets** — The block parser's assignment detection only handled `Ident` and `Deref` targets. Added `ExprKind::FieldAccess` arm to produce `StmtKind::AssignField`.

- **Integration test file corruption** — Multiple test insertions created brace mismatches, missing closing braces, and duplicate test functions. Required a complete rewrite of the integration test file to fix.

- **UI test snapshots had ANSI codes** — Added `std::env::set_var("NO_COLOR", "1")` in the UI test harness to disable color output for consistent snapshot comparison.

- **`SyntaxKind::COUNT` out of sync** — Adding variants (`Amp`, `KwFor`, `KwIn`, `WhileExpr`, `ForInExpr`, etc.) required updating the hardcoded `COUNT` and the corresponding test assertion. Fixed with automatic counting.

- **Git lock conflicts** — A stale `.git/index.lock` prevented `git checkout` from restoring broken files. Manually removed the lock.

## Decisions Made

- **Structural type‑param inference** — Instead of positional mapping, inference walks parameter types and matches the type variable symbol against the declared type, using the corresponding argument's inferred type. This correctly handles `fn wrap<T>(label: i64, value: T)`.

- **`fn_mangle_map` keyed by full specialization** — To avoid collisions when the same generic function is instantiated with different type arguments, the mangle map uses `(Symbol, Vec<HirType>)` as the key.

- **Three‑pass codegen** — Structs/enums/externs (`pass 1`), forward‑declare all functions (`pass 2`), emit bodies (`pass 3`). Ensures monomorphized functions are visible before any caller body is compiled.

- **Heap‑allocated structs** — Struct literals are now allocated on the heap to prevent use‑after‑free bugs when struct pointers escape the function scope.

- **`self: Type` as the canonical form** — `&self` sugar desugars to `self: TargetType` in the AST, keeping the HIR representation uniform.
