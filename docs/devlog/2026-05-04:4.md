## Problems Hit

- **Duplicate test functions from append scripts** — Multiple scripts appended the same test functions to `tests.rs`, causing "name defined multiple times" errors.  
  **Resolution:** Wrote Python deduplication scripts, then eventually rewrote the entire `tests.rs` from scratch with all tests exactly once.

- **Borrow‑checker issues in inline tests** — Tests like `bind_pattern_wild_and_ignore` tried to call `tc.lookup_binding(&tc.interner.intern("anything"))`, which borrows `tc` immutably and `tc.interner` mutably simultaneously.  
  **Resolution:** Interned the string into a local variable first, then used that variable in the lookup.

- **Privacy errors for `pub(crate)` methods** — Early attempts to write tests in `tests/unit/` failed because `push_scope`, `check_stmt`, `insert_binding`, `lookup_binding`, `check_expr`, `check_fn`, `bind_match_pattern`, and `fns` are all `pub(crate)`.  
  **Resolution:** Moved all tests requiring private access into `src/typeck/tests.rs` (inside the crate), where they have full access. Also created `src/typeck/tests_extra.rs` (later merged) for additional tests.

- **Production code breakage from test‑writing scripts** — At one point, a script accidentally modified `match_check.rs` and `expr.rs` (changing arm types and imports), causing cascading compilation errors across `glyim-typeck` and `glyim-codegen-llvm`.  
  **Resolution:** Used `git checkout` to restore the affected files, then added tests more carefully.

- **Snapshot test failure for new `enum_payload.g`** — The original snapshot source had a syntax error (comma between match arms).  
  **Resolution:** Simplified the test program to use wildcard patterns and unnamed enum variants, then accepted the new snapshot with `INSTA_UPDATE=always`.

- **Timeouts in mutation testing** — The new snapshot and integration tests increased test suite runtime, causing 31 mutants to timeout instead of producing a pass/fail result.  
  **Resolution:** Recommended increasing `--timeout-multiplier 3` for the next run, and consider moving slow tests out of the typeck crate.

- **Two newly added tests had incorrect assertions** — `bind_pattern_generic_struct` expected `RawPtr(Int)` but the actual field type remains `RawPtr(Named(T))` (type substitution hasn't occurred yet at bind time). `infer_generic_call_empty_type_args` expected `Int` but the return type is `Named(T)`.  
  **Resolution:** Corrected both assertions to match the actual current behavior.

## Decisions Made

- **Tests requiring private access go in `src/typeck/tests.rs`** — Rather than making internal APIs `pub`, all white‑box unit tests for the type checker live inside the crate in the `tests` module. This gives full access to `pub(crate)` methods and fields without expanding the public API surface.

- **One test file per concern, but all in‑crate** — All new type‑checker tests are in `src/typeck/tests.rs` (≈1400 lines). This is a single file for simplicity, but organized into clear sections (scope tests, inference tests, error tests, etc.). If it grows further, it can be split into `tests/` submodules within `src/typeck/`.

- **Snapshot tests for error messages are valuable** — Adding `insta` snapshot tests for `TypeError` lists gives us regression protection for the exact phrasing and structure of error messages, which is important for IDE integration and user experience.
