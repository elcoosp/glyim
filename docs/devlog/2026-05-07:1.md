  - Goto‑definition and find‑references use the symbol index and reference graph.
  - Document symbols returns a nested hierarchical list.

## Problems Hit

- **The LSP framework saga** — Selecting the right async LSP library was the single most time‑consuming obstacle. `tower-lsp` (0.20) worked but required `async_trait` and careful `Send` management (cloning diagnostics before `.await`). `tower-lsp-server` (community fork 0.23) had breaking changes in its `LanguageServer` trait and required `ls_types` (not `lsp_types`), leading to type mismatch nightmares. `async-lsp` appeared ideal but we fought through incorrect usage of `Router`, `MainLoop::new` vs `new_server`, missing `lock_tokio` (it’s `PipeStdin::lock_tokio`, not `lock`), and `log_message` vs `notify::<LogMessage>` confusion. After many iterations, the final implementation uses `MainLoop::new_server` with a builder closure that receives a `ClientSocket`, correctly registers notification and request handlers, and publishes diagnostics via `notify::<PublishDiagnostics>`.

- **Sed gone rogue** — An attempt to replace `HirItem::Fn(f)` with `HirItem::Fn(ref f)` to support a `/Box<HirFn>/` change corrupted multiple files (including backup files and test files) and required `git checkout` to restore purity. We ultimately reverted the boxing and kept `HirFn` as a direct variant.

- **Clippy warnings accumulation** — After each compilation fix, new warnings appeared, requiring iterative passes through `cargo fix` and manual edits. The `format_in_format_args` warning persisted across multiple attempts because the exact two‑line pattern differed subtly from the regex. We solved it by rewriting the file line‑by‑line.

## Decisions Made

- **No `Box<HirFn>`** — The clippy `large_enum_variant` warning was suppressed with `#[allow(...)]`. Boxing would have required hundreds of ref‑specializing changes across the codebase, outweighing the marginal memory savings.
- **Stick with `async-lsp` v0.2.4** — Despite the learning curve, its `MainLoop` architecture and explicit `ClientSocket` passing are well‑suited to our need to share the client across handlers.
- **Diagnostics pre‑publication** — To satisfy `Send` in notification handlers, we clone the diagnostic list while the lock is held, then drop the lock before calling `publish_diagnostics`. This avoids `RwLockReadGuard` crossing `.await`.
- **SourceMap is standalone** — Rather than integrating with `Span` immediately, we kept `Span` backward‑compatible and use `SourceMap` as a separate lookup structure. Full migration to `Span::file_id` can happen gradually.
- **Keep the mutation runner stub** — The `MutationRunner` parses and collects tests but marks all mutants as “survived” – this is sufficient for the CLI report format and scoring logic, while actual mutant compilation is deferred to later optimization.
