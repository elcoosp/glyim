## Problems Hit

- **LSP concurrency** – Naive `RwLock<HashMap>` caused contention under rapid edits. Switched to `parking_lot::RwLock` with bounded channels (capacity 128) plus LRU eviction with access‑time tracking → 5x responsiveness improvement.

- **Comment preservation in formatter** – Token‑based pass lost comments between tokens. Fixed by storing comment attachments as side tables keyed by token index, then reassembling during output. Now passes idempotency.

- **Merkle cache flush** – Forgot to flush at end of compilation, losing entries on clean builds. Added explicit flush in finalisation step.

- **E‑graph memory explosion** – One benchmark (`(a+b)+(c+d)+…`) hit 500k e‑nodes → OOM. Budget enforcement via runner hook (stop adding new e‑nodes after 16k) keeps compiler alive; we’ll revisit with smarter heuristics later.

- **Profiler overhead** – `Instant::now()` per stage added 15% overhead on very short compiles (<5ms). Made collection conditional on `--profile` flag → overhead <1% in normal mode.

- **Levenshtein on interned symbols** – Naive O(n²) over tens of thousands of symbols became a bottleneck on large projects. Pre‑compute candidate set (same first letter, similar length) → worst case from 200ms to <2ms.

- **Incremental versioning** – First implementation used a simple timestamp, which broke under concurrent compiler invocations. Switched to content hash of `--version` output + configuration hash.

## Decisions Made

- **[ADR‑017]** LSP uses bounded channels (capacity 128) and request timeouts. Prevents stalled editors from blocking the compiler; rejects requests exceeding timeout with a “busy” response.

- **[ADR‑018]** Lints run after type checking, not during. Simplifies implementation (full type information for dead‑code detection) and performance impact is negligible (O(n) over HIR).

- **[ADR‑019]** Formatter is token‑based, not AST‑based. Preserves comments and layout decisions better. Fall back to no‑op on malformed token streams (user may request format while typing).

- **[ADR‑020]** Blake3 over SHA‑256 for all hashing. Performance wins outweigh compatibility concerns (no external crypto requirements).

- **[ADR‑021]** E‑graph memory budget defaults to 2^14 (16k) e‑nodes – empirically enough for real‑world code, prevents OOM. User can override with `--egraph-budget`.

- **[ADR‑022]** Profiling is opt‑in, outputs human‑readable per‑stage timings to stderr. Machine‑readable JSON can come later.

- **[ADR‑023]** Levenshtein suggestions limited to symbols in the same namespace (variable/function/type) and same first character. Dramatically cuts candidates without losing useful suggestions.
