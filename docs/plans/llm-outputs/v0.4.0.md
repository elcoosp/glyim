## What the original plan got wrong

The original plan was written against a speculative v0.3.0 codebase that didn't match reality. Here's every error, so you can see what changed:

| Original plan claim | Actual codebase state | Impact |
|---|---|---|
| `node.rs` is ~55 LOC with 9 HirExpr variants | `node.rs` is ~120 LOC with **17** HirExpr variants (IntLit, FloatLit, BoolLit, StrLit, Ident, UnitLit, Binary, Unary, Block, If, Println, Assert, As, Match, FieldAccess, StructLit, EnumVariant) | Plan's `expr.rs` dropped 8 variants — would break compilation |
| `HirType`, `HirPattern`, `ExprId` don't exist yet | All three already exist in `types.rs` | Plan created duplicate definitions that would conflict |
| `HirItem`, `Hir`, struct/enum defs don't exist yet | All exist in `item.rs` and `node.rs` | Plan created duplicate types with different names (`HirStructDef` vs `StructDef`) |
| `lower()` takes `(&Ast, &Interner)` | Takes `(&Ast, &mut Interner)` — interner is mutated during lowering (TryExpr, SomeExpr, etc. intern new symbols) | Plan's replacement lower.rs would fail to compile |
| `hir.fns` field exists in `Hir` | Already `hir.items: Vec<HirItem>` | Plan's codegen change was already done |
| codegen iterates `hir.fns` | Already iterates `hir.items` with match on `HirItem::Fn/Struct/Enum/Extern` | Plan's codegen change was already done |
| `item.rs` doesn't exist | Already exists with `HirItem`, `StructDef`, `EnumDef`, `ExternBlock`, `ExternFn`, `FnSig`, `StructField`, `HirVariant` | Plan created a competing `item.rs` with different type names |
| `types.rs` doesn't exist | Already exists with `HirType` (11 variants), `HirPattern` (13 variants), `ExprId` (type alias) | Plan created duplicate types |
| Add `glyim-diag` dependency for `Span` in item types | `item.rs` has no `Span` fields; adding them is unnecessary for Phase 1 | Would add a dependency for no reason |
| Plan's `lower.rs` replacement handles 8 expression kinds | Actual `lower.rs` handles **22** expression kinds (all the v0.3.0 additions) | Plan's replacement silently dropped FloatLit, BoolLit, UnitLit, StructLit, Match, EnumVariant, FieldAccess, As, MacroCall, TryExpr, Some/None/Ok/Err, Pointer |
| `HirImplDef` needed | No parser support for impl blocks exists; adding the type now creates dead code | Defer to Phase 8 |
| Split into `node/` directory with 7 files | Files are all under 150 LOC; splitting adds navigation cost with no clarity gain | Optional — include as separate task |

- [ ] **Step 2: Add `parse_type_expr` method to Parser**

In `parser.rs`, add:
```rust
    fn parse_type_expr(&mut self) -> Option<TypeExpr> {
        match self.tokens.peek()?.kind {
            SyntaxKind::LParen => self.parse_tuple_type(),
            SyntaxKind::Star => self.parse_ptr_type(),
            SyntaxKind::Ident => self.parse_named_or_generic_type(),
            _ => None,
        }
    }

    fn parse_tuple_type(&mut self) -> Option<TypeExpr> {
        self.tokens.bump(); // consume '('
        if self.tokens.at(SyntaxKind::RParen) {
            self.tokens.bump();
            return Some(TypeExpr::Unit);
        }
        let mut elems = vec![];
        loop {
            elems.push(self.parse_type_expr()?);
            if !self.tokens.eat(SyntaxKind::Comma) {
                break;
            }
            if self.tokens.at(SyntaxKind::RParen) {
                break; // trailing comma
            }
        }
        self.tokens.expect(SyntaxKind::RParen).ok()?;
        Some(TypeExpr::Tuple(elems))
    }

    fn parse_ptr_type(&mut self) -> Option<TypeExpr> {
        self.tokens.bump(); // consume '*'
        let mutable = self.tokens.eat(SyntaxKind::KwMut).is_some();
        self.tokens.eat(SyntaxKind::KwLet); // skip 'const' keyword if present
        let inner = self.parse_type_expr()?;
        Some(TypeExpr::RawPtr { mutable, inner: Box::new(inner) })
    }

    fn parse_named_or_generic_type(&mut self) -> Option<TypeExpr> {
        let tok = self.tokens.bump()?;
        let sym = self.interner.intern(tok.text);
        let name = self.interner.resolve(sym).to_string();
        // Map primitive names
        match name.as_str() {
            "i64" | "Int" => return Some(TypeExpr::Int),
            "f64" | "Float" => return Some(TypeExpr::Float),
            "bool" | "Bool" => return Some(TypeExpr::Bool),
            "Str" | "str" | "String" => return Some(TypeExpr::Str),
            _ => {}
        }
        if self.tokens.at(SyntaxKind::Lt) {
            self.tokens.bump(); // consume '<'
            let mut args = vec![];
            loop {
                args.push(self.parse_type_expr()?);
                if self.tokens.at(SyntaxKind::Gt) {
                    self.tokens.bump();
                    break;
                }
                if !self.tokens.eat(SyntaxKind::Comma) {
                    break;
                }
            }
            Some(TypeExpr::Generic(sym, args))
        } else {
            Some(TypeExpr::Named(sym))
        }
    }
```

⚠️ **`<` vs `Lt` ambiguity:** The lexer produces `SyntaxKind::Lt` for `<`. In type argument position, `Lt` means "open generic", not "less than". The parser disambiguates contextually: when `parse_type_expr` is called, any `Lt` after a type name is treated as opening a generic argument list. This means you can't write `fn foo(x: i64) -> i64 where x < 10` — but `where` clauses are excluded from v0.4.0 anyway.

⚠️ **`>` vs `Gt` ambiguity:** Similarly, `Gt` in type position closes the generic. Nested `>>` (e.g., `Vec<Vec<Int>>`) requires `>>` to be two separate `Gt` tokens. The current lexer produces a single `Gt` for `>`, so `>>` is two `Gt` tokens. This works correctly.

```rust
fn push_scope(&mut self, type_params: Vec<Symbol>) {
    let parent = if self.scopes.is_empty() { None } else { Some(self.scopes.len() - 1) };
    let tp_map: HashMap<Symbol, ()> = type_params.into_iter().map(|s| (s, ())).collect();
    self.scopes.push(Scope { bindings: HashMap::new(), type_params: tp_map, parent });
}

fn pop_scope(&mut self) {
    self.scopes.pop();
}

fn lookup(&self, sym: Symbol) -> Option<HirType> {
    let mut idx = Some(self.scopes.len() - 1);
    while let Some(i) = idx {
        if let Some(ty) = self.scopes[i].bindings.get(&sym) {
            return Some(ty.clone());
        }
        idx = self.scopes[i].parent;
    }
    None
}

fn is_type_param(&self, sym: Symbol) -> bool {
    let mut idx = Some(self.scopes.len() - 1);
    while let Some(i) = idx {
        if self.scopes[i].type_params.contains_key(&sym) {
            return true;
        }
        idx = self.scopes[i].parent;
    }
    false
}
```

```rust
fn resolve_type(&self, ty: &HirType, subst: &HashMap<Symbol, HirType>) -> HirType {
    match ty {
        HirType::Int | HirType::Bool | HirType::Float | HirType::Str | HirType::Unit | HirType::Never => ty.clone(),
        HirType::Named(sym) => {
            if let Some(concrete) = subst.get(sym) {
                concrete.clone()
            } else {
                ty.clone()
            }
        }
        HirType::Generic(sym, args) => {
            let resolved_args: Vec<HirType> = args.iter().map(|a| self.resolve_type(a, subst)).collect();
            HirType::Generic(*sym, resolved_args)
        }
        HirType::Tuple(elems) => HirType::Tuple(elems.iter().map(|e| self.resolve_type(e, subst)).collect()),
        HirType::RawPtr { inner } => HirType::RawPtr { inner: Box::new(self.resolve_type(inner, subst)) },
        HirType::Option(inner) => HirType::Option(Box::new(self.resolve_type(inner, subst))),
        HirType::Result(t, e) => HirType::Result(
            Box::new(self.resolve_type(t, subst)),
            Box::new(self.resolve_type(e, subst)),
        ),
        HirType::Func(params, ret) => HirType::Func(
            params.iter().map(|p| self.resolve_type(p, subst)).collect(),
            Box::new(self.resolve_type(ret, subst)),
        ),
        HirType::Opaque(sym) => HirType::Opaque(*sym),
    }
}
```

```rust
        Pattern::Var(name) => HirStmt::Let { name, mutable, value: val },
        other => HirStmt::LetPat { pattern: lower_pattern(&other, interner), mutable, value: val },
    }
}
```

- [ ] **Step 2: Implement `check_let_pat`**

```rust
fn check_let_pat(&mut self, pattern: &HirPattern, value_ty: &HirType) {
    match pattern {
        HirPattern::Var(sym) => {
            self.scopes.last_mut().unwrap().bindings.insert(*sym, value_ty.clone());
        }
        HirPattern::Wild => {}
        HirPattern::Tuple { elements } => {
            if let HirType::Tuple(elem_types) = value_ty {
                for (pat, ty) in elements.iter().zip(elem_types.iter()) {
                    self.check_let_pat(pat, ty);
                }
            } else {
                self.errors.push(TypeError::MismatchedTypes { ... });
            }
        }
        HirPattern::Struct { name, bindings } => {
            if let Some(info) = self.structs.get(name) {
                for (field_name, field_pat) in bindings {
                    if let Some(&idx) = info.field_map.get(field_name) {
                        self.check_let_pat(field_pat, &info.fields[idx].ty);
                    }
                }
            }
        }
        HirPattern::EnumVariant { enum_name, variant_name, bindings } => {
            // Resolve enum, find variant, check bindings against variant field types
        }
        _ => {} // literals, unit — no bindings to extract
    }
}
```

- [ ] **Step 3: Update codegen for `LetPat`**

For struct destructuring: GEP each field, load, store into binding alloca.
For tuple destructuring: GEP each element by index, load, store.
For enum variant destructuring: check tag, GEP payload fields.

⚠️ **Codegen complexity:** This is the most complex codegen task. Each pattern type needs different LLVM IR generation. For v0.4.0, struct and tuple destructuring are required; enum variant destructuring in let can be stubbed (users can use match instead).

## Phase 8: Impl Blocks — Desugaring and Type Checking

**Prerequisites:** Phase 4 complete (impl methods reference struct types).

### Task 8.1: Desugar Impl Blocks to Namespaced Functions

⚠️ **Where to desugar?** Two options:
a) Desugar in `lower.rs` — impl methods become top-level `HirItem::Fn` with mangled names
b) Desugar in a separate pass between lowering and type checking

Option (a) is simpler and matches ADR-028. Do it in `lower.rs`:

```rust
Item::ImplBlock { target, type_params, methods, .. } => {
    for method in methods {
        if let Item::FnDef { name, type_params: fn_tp, params, ret, body, .. } = method {
            let mangled_name = interner.intern(&format!(
                "{}_{}", interner.resolve(*target), interner.resolve(*name)
            ));
            let all_tp: Vec<Symbol> = type_params.iter().chain(fn_tp.iter()).copied().collect();
            // Prepend self parameter if first param is "self"
            let hir_params = /* ... */;
            fns.push(HirItem::Fn(HirFn {
                name: mangled_name,
                type_params: all_tp,
                params: hir_params,
                ret: hir_ret,
                body: lower_expr_with_id(&body.kind, interner, &mut fresh_id),
            }));
        }
    }
}
```

⚠️ **`self` parameter handling:** Per spec, `self` is the first parameter typed as the impl target. In `fn translate(self: Point, dx: i64)`, `self` is just a regular parameter named `self` with type `Point`. No special AST node needed — it's syntactic sugar for `fn translate(self: Point, ...)` where `self` has the impl target type.

### Task 8.2: Call Site Desugaring

`p.translate(3)` must become `Point_translate(p, 3)`.

⚠️ **Current limitation:** The parser handles `p.translate(3)` as `ExprKind::Call { callee: ExprKind::FieldAccess { object: p, field: "translate" }, args: [3] }`. The type checker needs to:
1. Type-check `p` → `Point`
2. Look up impl methods on `Point` → find `Point_translate`
3. Rewrite the call to `Point_translate(p, 3)`
4. Type-check with the rewritten callee

This requires the type checker to maintain an impl method table:
```rust
impl_methods: HashMap<Symbol, Vec<Symbol>>,  // struct_name → [method_names]
```

### Task 10.1: Cast Validity Checking

- [ ] **Step 1: Define valid casts**

```rust
fn is_valid_cast(from: &HirType, to: &HirType) -> bool {
    match (from, to) {
        // Numeric ↔ Numeric
        (HirType::Int, HirType::Float) | (HirType::Float, HirType::Int) => true,
        (HirType::Int, HirType::Int) | (HirType::Float, HirType::Float) => true,
        // Any → RawPtr
        (_, HirType::RawPtr { .. }) => true,
        // RawPtr → RawPtr
        (HirType::RawPtr { .. }, HirType::RawPtr { .. }) => true,
        // Str → RawPtr (string data pointer)
        (HirType::Str, HirType::RawPtr { .. }) => true,
        // Same type (no-op cast)
        (a, b) if a == b => true,
        _ => false,
    }
}
```

- [ ] **Step 2: Check in `check_expr`**

```rust
HirExpr::As { expr, target_type, id, span } => {
    let from_ty = self.check_expr(expr)?;
    if !is_valid_cast(&from_ty, target_type) {
        self.errors.push(TypeError::InvalidCast {
            from: from_ty,
            to: target_type.clone(),
            span: Some(*span),
        });
    }
    self.set_type(*id, target_type.clone());
    Some(target_type.clone())
}
```

### Task 10.2: Cast Codegen

- [ ] **Step 1: Implement numeric casts**

```rust
HirExpr::As { expr, target_type, .. } => {
    let val = self.codegen_expr(expr, ...)?;
    match target_type {
        HirType::Float => {
            self.builder.build_si_to_fp(val, self.f64_type, "sitofp").ok()
                .map(|v| self.builder.build_bitcast(v, self.i64_type, "as_i64").ok())
                .flatten()
        }
        HirType::Int => {
            let fp_val = self.builder.build_bitcast(val, self.f64_type, "as_f64").ok()?;
            self.builder.build_fp_to_si(fp_val, self.i64_type, "fptosi").ok()
                .map(|v| v)
        }
        HirType::RawPtr { .. } => {
            self.builder.build_int_to_ptr(val, self.context.ptr_type(AddressSpace::from(0u16)), "ptr").ok()
                .map(|v| self.builder.build_ptr_to_int(v, self.i64_type, "as_int").ok())
                .flatten()
        }
        _ => Some(val), // no-op or stub
    }
}
```

### Task 11.2: Monomorphization Pass

⚠️ **Scope decision:** The spec describes a full monomorphization pass that replaces generic functions with concrete monomorphs. For v0.4.0, a simpler approach works: monomorphize at codegen time. When codegen encounters a call to a generic function, it looks up the inferred type args, substitutes them, and generates the monomorph inline. This avoids a separate HIR pass.

- [ ] **Step 1: Add monomorph cache to Codegen**

```rust
mono_cache: RefCell<HashMap<(Symbol, Vec<HirType>), FunctionValue<'ctx>>>,
```

- [ ] **Step 2: When codegen encounters a call to a generic function:**

1. Look up the function's type_params and body from the HIR
2. Get the concrete type args from the type map (using the callee expr's id)
3. Check mono_cache for existing monomorph
4. If not cached: substitute type params in the body, recursively codegen the substituted body as a new LLVM function
5. Call the monomorphized function

⚠️ **Type substitution in HIR:** Need a function `subst_expr(expr: &HirExpr, subst: &HashMap<Symbol, HirType>) -> HirExpr` that walks the expression and replaces `HirType::Generic(sym, [])` with the concrete type from `subst`. This does NOT change the expression structure — only the types embedded in `As`, `StructLit` field types, etc.
