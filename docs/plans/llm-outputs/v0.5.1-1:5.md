# Phase 1: HIR Spans & Per-Function DWARF — Detailed Implementation

## Overview

This phase adds source location tracking to every HIR node, then creates a DWARF debug info system from scratch (no prior debug infrastructure exists). The result: compiled binaries carry function names, source lines, and instruction-level locations visible in `gdb`/`lldb`.

## Chunk 2: Propagate Spans Through Lowering

### Task 8: Update `lower_expr` to carry and propagate spans

**File:** `crates/glyim-hir/src/lower/expr.rs`

The current signature is:
```rust
pub fn lower_expr(expr: &ExprKind, ctx: &mut LoweringContext) -> HirExpr
```

Change to:
```rust
pub fn lower_expr(expr: &ExprKind, span: Span, ctx: &mut LoweringContext) -> HirExpr
```

Now thread `span` into every variant:

```rust
pub fn lower_expr(expr: &ExprKind, span: Span, ctx: &mut LoweringContext) -> HirExpr {
    let id = ctx.fresh_id();
    match expr {
        ExprKind::IntLit(n) => HirExpr::IntLit { id, value: *n, span },
        ExprKind::FloatLit(f) => HirExpr::FloatLit { id, value: *f, span },
        ExprKind::BoolLit(b) => HirExpr::BoolLit { id, value: *b, span },
        ExprKind::StrLit(s) => HirExpr::StrLit { id, value: s.clone(), span },
        ExprKind::Ident(sym) => HirExpr::Ident { id, name: *sym, span },
        ExprKind::UnitLit => HirExpr::UnitLit { id, span },

        ExprKind::Binary { op, lhs, rhs } => {
            let lhs_span = /* need the lhs ExprNode's span */;
            let rhs_span = /* need the rhs ExprNode's span */;
            HirExpr::Binary {
                id,
                op: lower_binop(op.clone()),
                lhs: Box::new(lower_expr(&lhs.kind, lhs_span, ctx)),
                rhs: Box::new(lower_expr(&rhs.kind, rhs_span, ctx)),
                span,  // covers the entire binary expression
            }
        }
        // ... etc
    }
}
```

**Problem:** `lower_expr` receives `&ExprKind` (the enum interior), not `&ExprNode` (which has the span). The callers have the `ExprNode` with the span. We need to change the call sites to pass `expr_node.span`.

### Task 9: Update all `lower_expr` call sites to pass spans

**File:** `crates/glyim-hir/src/lower/expr.rs`

Every recursive call to `lower_expr` must pass the appropriate span. The key cases:

**From `lower_block`:**
```rust
fn lower_block(items: &[BlockItem], ctx: &mut LoweringContext) -> HirExpr {
    // We need the block's span here. Pass it in from the caller.
    // Change signature: lower_block(items, block_span, ctx)
}
```

**From binary/recursive expressions:**
```rust
ExprKind::Binary { op, lhs, rhs } => {
    // Problem: we have &ExprKind, not &ExprNode, so we don't have lhs.span/rhs.span
    // Solution: change lower_expr to take &ExprNode instead of &ExprKind
}
```

**Better approach:** Change `lower_expr` to take `&ExprNode`:

```rust
pub fn lower_expr(expr: &ExprNode, ctx: &mut LoweringContext) -> HirExpr {
    let id = ctx.fresh_id();
    let span = expr.span;
    match &expr.kind {
        ExprKind::IntLit(n) => HirExpr::IntLit { id, value: *n, span },
        ExprKind::Binary { op, lhs, rhs } => {
            HirExpr::Binary {
                id,
                op: lower_binop(op.clone()),
                lhs: Box::new(lower_expr(lhs, ctx)),
                rhs: Box::new(lower_expr(rhs, ctx)),
                span,
            }
        }
        ExprKind::If { condition, then_branch, else_branch } => {
            HirExpr::If {
                id,
                condition: Box::new(lower_expr(condition, ctx)),
                then_branch: Box::new(lower_expr(then_branch, ctx)),
                else_branch: else_branch.as_ref().map(|e| Box::new(lower_expr(e, ctx))),
                span,
            }
        }
        // ... every other variant: use expr.span for the outer, recurse with sub-ExprNodes
        ExprKind::Block(items) => lower_block_expr(items, span, ctx),
        ExprKind::Match { scrutinee, arms } => lower_match_expr(id, scrutinee, arms, span, ctx),
        // For Some/Ok/Err sugar — the span comes from the outer ExprNode
        ExprKind::SomeExpr(e) => HirExpr::EnumVariant {
            id,
            enum_name: ctx.intern("Option"),
            variant_name: ctx.intern("Some"),
            args: vec![lower_expr(e, ctx)],
            span,
        },
        // ... etc
    }
}
```

This is cleaner because every `ExprNode` already carries its `span: Span`.

### Task 10: Update `lower_block` to accept and use span

```rust
fn lower_block_expr(items: &[BlockItem], block_span: Span, ctx: &mut LoweringContext) -> HirExpr {
    let id = ctx.fresh_id();
    let stmts: Vec<HirStmt> = items
        .iter()
        .map(|item| match item {
            BlockItem::Expr(e) => HirStmt::Expr(lower_expr(e, ctx)),
            BlockItem::Stmt(s) => lower_stmt(s, ctx),
        })
        .collect();
    HirExpr::Block { id, stmts, span: block_span }
}
```

### Task 11: Update `lower_stmt` to produce spans

```rust
fn lower_stmt(stmt: &glyim_parse::StmtNode, ctx: &mut LoweringContext) -> HirStmt {
    let span = stmt.span;
    match &stmt.kind {
        StmtKind::Let { pattern, mutable, value } => {
            let val = lower_expr(value, ctx);
            match pattern {
                Pattern::Var(name) => HirStmt::Let {
                    name: *name,
                    mutable: *mutable,
                    value: val,
                    span,
                },
                _ => HirStmt::Let {
                    name: ctx.intern("_"),
                    mutable: false,
                    value: val,
                    span,
                },
            }
        }
        StmtKind::Assign { target, value } => HirStmt::Assign {
            target: *target,
            value: lower_expr(value, ctx),
            span,
        },
    }
}
```

### Task 12: Update `lower_match` to produce spans

```rust
fn lower_match_expr(
    id: ExprId,
    scrutinee: &glyim_parse::ExprNode,
    arms: &[glyim_parse::MatchArm],
    match_span: Span,
    ctx: &mut LoweringContext,
) -> HirExpr {
    let hir_arms: Vec<(HirPattern, Option<HirExpr>, HirExpr)> = arms
        .iter()
        .map(|arm| {
            let pattern = lower_pattern(&arm.pattern, ctx);
            let guard = arm.guard.as_ref().map(|e| lower_expr(e, ctx));
            let body = lower_expr(&arm.body, ctx);
            (pattern, guard, body)
        })
        .collect();
    HirExpr::Match {
        id,
        scrutinee: Box::new(lower_expr(scrutinee, ctx)),
        arms: hir_arms,
        span: match_span,
    }
}
```

### Task 14: Update `lower_pattern` for compound patterns

**File:** `crates/glyim-hir/src/lower/pattern.rs`

The `Struct` and `EnumVariant` and `Tuple` pattern variants need spans. But `lower_pattern` receives `&Pattern` (not an AST node with a span). We need to either:

**(a)** Add spans to AST `Pattern` (in `glyim-parse/src/ast.rs`), or  
**(b)** Pass the span down from the call site.

Option (b) is simpler for now — add an optional `span` parameter to `lower_pattern`:

```rust
pub fn lower_pattern(pat: &Pattern, ctx: &mut LoweringContext) -> HirPattern {
    lower_pattern_with_span(pat, None, ctx)
}

fn lower_pattern_with_span(pat: &Pattern, span: Option<Span>, ctx: &mut LoweringContext) -> HirPattern {
    match pat {
        // ... leaf patterns unchanged
        Pattern::Struct { name, fields } => {
            let s = span.unwrap_or(Span::new(0, 0));
            HirPattern::Struct {
                name: *name,
                bindings: fields.iter().map(|(sym, p)| (*sym, lower_pattern_with_span(p, None, ctx))).collect(),
                span: s,
            }
        }
        Pattern::EnumVariant { enum_name, variant_name, args } => {
            let s = span.unwrap_or(Span::new(0, 0));
            HirPattern::EnumVariant {
                enum_name: *enum_name,
                variant_name: *variant_name,
                bindings: args.iter().enumerate().map(|(i, p)| (ctx.intern(&i.to_string()), lower_pattern_with_span(p, None, ctx))).collect(),
                span: s,
            }
        }
        Pattern::Tuple(elems) => {
            let s = span.unwrap_or(Span::new(0, 0));
            HirPattern::Tuple {
                elements: elems.iter().map(|e| lower_pattern_with_span(e, None, ctx)).collect(),
                span: s,
            }
        }
        // ... all other patterns: no span field, return as-is
    }
}
```

The call site in `lower_match` can pass `Some(arm.pattern_span)` if the AST carry pattern spans. Currently `MatchArm` doesn't have a pattern span — the span would need to be inferred from the first token of the pattern to the `=>`. For now, use `None` (produces `Span::new(0,0)` placeholder).

### Task 25: Implement DebugInfoGen::new()

This is the hardest part of the entire phase. The inkwell API for creating debug info is finicky. Here's the best guess based on inkwell 0.5–0.9 patterns:

```rust
impl<'ctx> DebugInfoGen<'ctx> {
    pub fn new(module: &Module<'ctx>, file_name: &str) -> Result<Self, String> {
        let dibuilder = module.create_debug_info_builder(
            false,  // allow_unresolved
            inkwell::debug_info::DWARFEmissionKind::Full,
        ).map_err(|e| format!("create_debug_info_builder failed: {e}"))?;

        // Create a file — the API may vary
        let file = dibuilder.create_file(
            file_name,
            ".",  // directory
        );

        // Create compile unit — the API signature varies significantly between inkwell versions
        let compile_unit = dibuilder.create_compile_unit(
            file,
            file_name,
            "glyim v0.5.1",
            false,  // optimized
            "",     // flags
            0,      // runtime version
            // There may be additional required parameters — check inkwell docs
        ).map_err(|e| format!("create_compile_unit failed: {e}"))?;

        Ok(Self {
            dibuilder,
            compile_unit,
            file,
            fn_subprograms: RefCell::new(HashMap::new()),
        })
    }
```

```rust
pub fn compile_to_ir_debug(source: &str, enable_debug: bool) -> Result<String, String> {
    let out = glyim_parse::parse(source);

    } else {

    };
    cg.generate(&hir)?;
    Ok(cg.ir_string())
}
```

```rust
    let void_type = module.get_context().void_type();
```

## Chunk 1: Allocator Wrapper Functions

### Task 1: Rewrite `emit_alloc_shims` signature and body

**File:** `crates/glyim-codegen-llvm/src/alloc.rs`

Replace the entire function. The new version:
- Takes `no_std: bool` parameter — if true, emits nothing
- Declares `malloc`, `free`, and `abort` as external functions
- Defines `glyim_alloc(size: i64) -> *i8` that calls `malloc`, checks for null, aborts on OOM, returns the pointer
- Defines `glyim_free(ptr: *i8) -> void` that calls `free`

```rust
/// Emit `glyim_alloc` and `glyim_free` wrapper functions.
///
/// `glyim_alloc(size: i64) -> *i8`:
///   Calls malloc(size). If malloc returns null, calls abort().
///   Otherwise returns the pointer.
///
/// `glyim_free(ptr: *i8) -> void`:
///   Calls free(ptr).
///
/// If `no_std` is true, emits nothing — no_std programs manage their own memory.
pub fn emit_alloc_shims(module: &inkwell::module::Module<'_>, no_std: bool) {
    if no_std {
        return;
    }

    let ctx = module.get_context();
    let i8_type = ctx.i8_type();
    let i64_type = ctx.i64_type();
    let void_type = ctx.void_type();
    let ptr_type = i8_type.ptr_type(inkwell::AddressSpace::from(0u16));

    // ── Declare external functions ──────────────────────────────

    // malloc(size: i64) -> *i8
    let malloc_ty = ptr_type.fn_type(&[i64_type.into()], false);
    let malloc = module.add_function("malloc", malloc_ty, None);

    // free(ptr: *i8) -> void
    let free_ty = void_type.fn_type(&[ptr_type.into()], false);
    let free_fn = module.add_function("free", free_ty, None);

    // abort() -> void
    let abort_ty = void_type.fn_type(&[], false);
    let abort_fn = module.add_function("abort", abort_ty, None);

    // ── glyim_alloc(size: i64) -> *i8 ───────────────────────────

    let alloc_fn_type = ptr_type.fn_type(&[i64_type.into()], false);
    let alloc_fn = module.add_function("glyim_alloc", alloc_fn_type, None);

    let entry = ctx.append_basic_block(alloc_fn, "entry");
    let oom_block = ctx.append_basic_block(alloc_fn, "oom");
    let ok_block = ctx.append_basic_block(alloc_fn, "ok");

    let builder = ctx.create_builder();

    // entry: call malloc, compare result to null
    builder.position_at_end(entry);
    let size_param = alloc_fn.get_first_param().unwrap();
    let raw_ptr = builder
        .build_call(malloc, &[size_param.into()], "raw_ptr")
        .unwrap()
        .try_as_basic_value()
        .left()
        .unwrap()
        .into_pointer_value();

    let null_ptr = ptr_type.const_null();
    let is_null = builder
        .build_int_compare(
            inkwell::IntPredicate::EQ,
            raw_ptr,
            null_ptr,
            "is_null",
        )
        .unwrap();

    builder
        .build_conditional_branch(is_null, oom_block, ok_block)
        .unwrap();

    // oom: call abort(), then unreachable
    builder.position_at_end(oom_block);
    builder.build_call(abort_fn, &[], "").unwrap();
    builder.build_unreachable().unwrap();

    // ok: return the pointer
    builder.position_at_end(ok_block);
    builder.build_return(Some(&raw_ptr)).unwrap();

    // ── glyim_free(ptr: *i8) -> void ─────────────────────────────

    let free_wrapper_type = void_type.fn_type(&[ptr_type.into()], false);
    let free_wrapper = module.add_function("glyim_free", free_wrapper_type, None);
    let free_entry = ctx.append_basic_block(free_wrapper, "entry");

    builder.position_at_end(free_entry);
    let ptr_param = free_wrapper.get_first_param().unwrap();
    builder.build_call(free_fn, &[ptr_param.into()], "").unwrap();
    builder.build_return(None).unwrap();
}
```

## Chunk 2: #[no_std] Detection

### Task 9: Understand the no_std design decision

In Rust, `#[no_std]` is a crate-level attribute. In Glyim, there's no attribute system for this yet. The design choice for v0.5.1 is: **a bare `no_std` identifier on its own line acts as a declaration** that the program should not receive the standard allocator or standard library prelude.

This is intentionally simple — a future version could add proper `#[no_std]` attribute syntax.

**Constraints:**
- Must not false-positive on `no_std` appearing inside strings: `"no_std"` → should NOT trigger
- Must not false-positive on `no_std` as part of a larger identifier: `no_std_helper` → should NOT trigger
- Must not false-positive on `no_std` inside comments — but this is acceptable for v0.5.1 since comments aren't stripped before this check
- A bare `no_std` with trailing whitespace should trigger: `no_std   ` → YES

**Where does detection happen?** In the CLI pipeline, before creating the `Codegen` instance. The detection result is passed as a flag.

---

### Task 10: Implement `detect_no_std`

**File:** `crates/glyim-cli/src/pipeline.rs`

Add a new free function near the top of the file, before the `PRELUDE` constant:

```rust
/// Detect whether the source contains a bare `no_std` declaration.
///
/// A `no_std` declaration is a line where `no_std` appears as a standalone
/// identifier — not part of a larger word, not inside a string literal.
///
/// Known limitation: does not filter out `no_std` inside comments.
/// This is acceptable for v0.5.1 — the full parser handles comments correctly.
fn detect_no_std(source: &str) -> bool {
    for line in source.lines() {
        let trimmed = line.trim();
        if trimmed == "no_std" {
            return true;
        }
    }
    false
}
```

**Why not parse properly?** The full parser (`glyim_parse::parse`) already runs later and strips comments. Running it twice just for `no_std` detection would be wasteful. The simple line-based check is sufficient because:
- `no_std` is extremely unlikely to appear as a false positive in real code
- The consequence of a false positive is minor: allocator shims aren't emitted (the program just can't use heap allocation)
- Comments containing `no_std` are an edge case we document

# Phase 3: Standard Library Core (Vec, String, Iterator, Range) — Detailed Implementation

## Critical Reality Assessment

Before writing any code, we must assess what the Glyim language at v0.3.0 can actually compile:

| Feature Needed | Status | Impact |
|---|---|---|
| Generic `*mut T` pointer | ❌ Only `*mut u8`/`*const u8` with named types | Vec<T> can't index into generic-typed memory |
| Pointer read/write through `*ptr` | ❌ All values are i64; no `load`/`store` on pointer-typed values | No way to dereference a pointer in Glyim |
| `size_of::<T>()` | ❌ No builtin | Needed for Vec growth calculation |
| `memcpy` | ❌ No builtin | Needed for Vec buffer reallocation |
| `null_mut()` | ❌ No builtin | Needed for Vec::new() |
| `impl Trait for Type` method resolution | ❌ Parses but call-site `Type::method()` doesn't resolve mangled names | Iterator trait can't be used with `for ... in` |
| Destructor/Drop | ❌ No concept | Vec memory leaks when it goes out of scope |
| `for i in range` syntax | ❌ No syntax | Range only usable with manual `loop`/`match` |

**Conclusion:** A stdlib written purely in Glyim source code cannot be compiled by the current compiler. The pointer operations required for `Vec<T>` are fundamentally impossible when all values are represented as `i64`.

**Revised Phase 3 strategy:**

1. **Add `size_of` as a compiler builtin** — this IS feasible (emits LLVM `sizeof`)
2. **Extend the inline prelude** with `String` type representation
3. **Create stdlib source files** as API documentation/specification that compiles when the language gains pointer operations
4. **Wire prelude file loading** with fallback to inline
5. **Write Vec/Iterator/Range** as uncompilable spec files with `// TODO` annotations
6. **Document precisely** what blocks each type

```glyim
// Glyim Standard Library — Vec<T>
//
// A growable array type backed by heap allocation.
//
// STATUS: Cannot be compiled by glyim v0.5.1
// BLOCKERS:
//   §1. Generic *mut T not supported — would need *mut u8 with unsafe casts
//   §2. Pointer load/store (dereferencing) not supported for generic types
//   §3. __size_of::<T>() works but pointer arithmetic on *mut u8 needs
//      byte offsets computed manually: ptr + (index * __size_of::<T>())
//   §4. No Drop/destructor — Vec memory leaks when it goes out of scope
//   §5. Struct method call resolution broken (Vec::push can't be called as v.push())
//
// DESIGN (for when blockers are resolved):
//   - Internal representation: struct Vec<T> { data: *mut u8, len: i64, cap: i64 }
//   - All pointer math done in bytes using *mut u8, with manual size_of multiplication
//   - Growth factor: 2x (or 8 for initial allocation)
//   - OOM: glyim_alloc returns null → abort (handled by wrapper)
//   - Drop: calls glyim_free on data pointer

// struct Vec<T> {
//     data: *mut u8,    // pointer to heap buffer (byte-addressable)
//     len: i64,          // number of elements (not bytes)
//     cap: i64,          // capacity in elements
// }
//
// fn Vec::new<T>() -> Vec<T> {
//     Vec { data: null_mut::<u8>(), len: 0, cap: 0 }
// }
//
// fn Vec::push<T>(self: *mut Vec<T>, value: T) {
//     if self.len == self.cap {
//         let new_cap = if self.cap == 0 { 8 } else { self.cap * 2 };
//         let byte_size = new_cap * __size_of::<T>();
//         let new_data = glyim_alloc(byte_size);
//         if self.len > 0 {
//             memcpy(new_data, self.data, self.len * __size_of::<T>());
//         }
//         if self.cap > 0 {
//             glyim_free(self.data);
//         }
//         self.data = new_data;
//         self.cap = new_cap;
//     }
//     let byte_offset = self.len * __size_of::<T>();
//     let ptr = self.data + byte_offset;  // pointer arithmetic on *mut u8
//     *(ptr as *mut T) = value;           // write through cast pointer
//     self.len = self.len + 1;
// }
//
// fn Vec::pop<T>(self: *mut Vec<T>) -> Option<T> {
//     if self.len == 0 { return None; }
//     self.len = self.len - 1;
//     let byte_offset = self.len * __size_of::<T>();
//     let ptr = self.data + byte_offset;
//     let value = *(ptr as *mut T);     // read through cast pointer
//     Some(value)
// }
//
// fn Vec::get<T>(self: *Vec<T>, index: i64) -> Option<T> {
//     if index < 0 || index >= self.len { return None; }
//     let byte_offset = index * __size_of::<T>();
//     let ptr = self.data + byte_offset;
//     Some(*(ptr as *mut T))
// }
//
// fn Vec::len<T>(self: *Vec<T>) -> i64 { self.len }
//
// fn Vec::is_empty<T>(self: *Vec<T>) -> bool { self.len == 0 }
//
// fn Vec::drop<T>(self: *mut Vec<T>) {
//     if self.cap > 0 {
//         glyim_free(self.data);
//         self.data = null_mut::<u8>();
//         self.len = 0;
//         self.cap = 0;
//     }
// }
//
// fn Vec::capacity<T>(self: *Vec<T>) -> i64 { self.cap }
```

## Chunk 5: Prelude Wiring (Deferred)

### Task 20: Document why prelude file loading is deferred

The prior draft had a `load_prelude()` function that read from `~/.glyim/std/0.5.1/src/prelude.g` with a fallback to `include_str!`. This adds complexity for zero user-visible benefit because:

1. The prelude file (`prelude.g`) only contains `use` statements for types that don't exist yet
2. The actual prelude content (Option/Result enum definitions) must stay inline because it's Glyim source that the compiler parses
3. Loading a file that just says `use vec::Vec` for a non-existent module adds confusion

**Decision:** Defer prelude file loading to when stdlib types are actually compilable. The inline `PRELUDE` constant in `pipeline.rs` remains the authoritative prelude. Add a comment documenting the future plan:

```rust
    Ok(T),
```

```rust
    InvalidArgument,

    Custom(i64),

```

This is different from the spec's `Error { message: String }` design, but it's actually usable now. Users can pattern-match on it and get meaningful error discrimination.

```rust
use serde::{Deserialize, Serialize};
use std::collections::HashMap;

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub struct Lockfile {
    #[serde(rename = "package")]
    pub packages: Vec<LockedPackage>,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub struct LockedPackage {
    pub name: String,
    pub version: String,
    pub hash: String,
    #[serde(default)]
    pub is_macro: bool,
    pub source: LockSource,
    #[serde(default)]
    pub deps: Vec<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
#[serde(tag = "type")]
pub enum LockSource {
    #[serde(rename = "registry")]
    Registry { url: String },
    #[serde(rename = "path")]
    Path { path: String },
    #[serde(rename = "local")]
    Local,
}

/// Generate a lockfile from resolved dependencies.
pub fn generate_lockfile(
    resolved: &HashMap<String, (String, String, bool, Vec<String>, LockSource)>,
) -> Lockfile {
    let mut packages: Vec<LockedPackage> = resolved
        .iter()
        .map(|(name, (version, hash, is_macro, deps, source))| LockedPackage {
            let mut deps = deps.clone();
            deps.sort();
            LockedPackage {
                name: name.clone(),
                version: version.clone(),
                hash: hash.clone(),
                is_macro: *is_macro,
                source: source.clone(),
                deps,
            }
        })
        .collect();
    packages.sort_by(|a, b| a.name.cmp(&b.name));
    Lockfile { packages }
}

/// Parse a lockfile from TOML string.
pub fn parse_lockfile(toml_str: &str) -> Result<Lockfile, PkgError> {
    let lockfile: Lockfile = toml::from_str(toml_str).map_err(|e| {
        PkgError::Lockfile(format!("invalid lockfile TOML: {e}"))
    })?;
    Ok(lockfile)
}

/// Serialize a lockfile to TOML string with header comment.
pub fn serialize_lockfile(lockfile: &Lockfile) -> String {
    let mut out = String::from(
        "# This file is @generated by glyim-pkg.\n\
# Do not edit manually.\n\n",
    );
    out.push_str(&toml::to_string_pretty(lockfile).unwrap_or_default());
    out
}

/// Compute a simple hash of byte content.
pub fn compute_content_hash(content: &[u8]) -> String {
    use sha2::{Digest, Sha256};
    let result = Sha256::digest(content);
    hex::encode(result)
}
```

### Task 16: Implement version constraint checking

```rust
/// Check if a concrete version satisfies a constraint string.
///
/// Supports:
///   - Exact: "1.2.3" → true only for "1.2.3"
///   - Caret: "^1.2.3" → true for ">=1.2.3, <2.0.0"
///   - Wildcard: "*" → true for everything
pub fn satisfies_constraint(version: &str, constraint: &str) -> bool {
    if constraint == "*" {
        return true;
    }
    if version == constraint {
        return true;
    }
    if let Some(rest) = constraint.strip_prefix("^") {
        let req_ver = rest;
        match semver::Version::parse(version).ok() {
            let rv = semver::Version::parse(req_ver).unwrap();
            // ^1.2.3 means >=1.2.3, <2.0.0
            return rv >= rv && rv.major == rv.major;
        }
        // If parse fails, fall through to exact match (already handled above)
    }
    false
}

/// Extract the major version from a version string for caret range check fallback.
fn major_version(version: &str) -> Option<u64> {
    semver::Version::parse(version).ok().map(|v| v.major)
}
```

### Task 17: Implement minimal version selection

```rust
/// Resolve dependencies using minimal version selection.
///
/// For each dependency:
/// 1. If in lockfile: use locked version
/// 2. If local path dependency: no resolution needed (path is the resolution)
/// 3. If registry dependency without lockfile: resolve fails (no registry yet)
pub fn resolve(
    root_deps: &[Requirement],
    lockfile: Option<&crate::lockfile::Lockfile>,
    available: &HashMap<String, Vec<AvailableVersion>>,
) -> Result<Resolution, PkgError> {
    let mut resolved: HashMap<String, ResolvedPackage> = HashMap::new();
    let mut constraints: HashMap<String, Vec<String>> = HashMap::new();

    // Seed with root dependencies
    for req in root_deps {
        constraints.entry(req.name.clone())
            .or_default()
            .push(req.version_constraint.clone());
    }

    let mut queue: Vec<String> = root_deps.iter().map(|r| r.name.clone()).collect();

    while let Some(name) = queue.pop() {
        if resolved.contains_key(&name) {
            continue;
        }

        // Check lockfile first
        if let Some(lock) = lockfile {
            if let Some(pkg) = lock.find_package_by_name(&name) {
                resolved.insert(name.clone(), ResolvedPackage {
                    version: pkg.version.clone(),
                    is_macro: pkg.is_macro,
                    deps: pkg.deps.clone(),
                    source: pkg.source.clone(),
                });
                // Queue transitive dependencies
                for dep in &pkg.deps {
                    if !resolved.contains_key(dep) {
                        constraints.entry(dep.clone())
                            .or_default()
                            .push("*"); // lockfile doesn't store constraints
                        queue.push(dep.clone());
                    }
                }
                continue;
            }
        }

        let versions = available.get(&name)
            .ok_or_else(|| {
                return Err(PkgError::Resolution(format!(
                    "package '{name}' not found in available packages"
                )));
            });

        let name_constraints = constraints
            .get(&name)
            .ok_or_else(|| {
                return Err(PkgError::Resolution(format!(
                    "no constraints for '{name}'"
                )));
            });

        let selected = find_min_satisfying(versions, name_constraints)?;

        for dep in &selected.deps {
            if !resolved.contains_key(dep) {
                constraints.entry(dep.clone())
                    .or_default()
                    .push("*");
                queue.push(dep.clone());
            }
        }

        resolved.insert(name.clone(), ResolvedPackage {
            version: selected.version.clone(),
            is_macro: selected.is_macro,
            deps: selected.deps.clone(),
            source: selected.source.clone(),
        });
    }

    Ok(Resolution { packages: resolved })
}

/// Find the minimum version in `available` that satisfies all constraints.
fn find_min_satisfying(
    versions: &[AvailableVersion],
    constraints: &[String],
) -> Option<&AvailableVersion> {
    versions
        .iter()
        .filter(|v| constraints.iter().all(|c| satisfies_constraint(&v.version, c)))
        .min_by(|a, b| {
            let va = semver::Version::parse(&a.version).unwrap();
            let vb = semver::Version::parse(&b.version).unwrap();
            va.cmp(&vb) == std::cmp::Ordering::Less
        })
}

/// An available version from the registry.
pub struct AvailableVersion {
    pub version: String,
    pub is_macro: bool,
    pub deps: Vec<Requirement>,
    pub source: LockSource,
}
```

### Task 22: Implement workspace detection

**File:** `crates/glyim-pkg/src/workspace.rs`

```rust
use crate::manifest::load_manifest;
use crate::error::PkgError;
use std::path::{Path, PathBuf};

pub struct Workspace {
    pub root: PathBuf,
    pub members: Vec<PathBuf>,
    pub manifest: crate::manifest::PackageManifest,
}

/// Detect if `start` is within a workspace by walking up looking for
/// a glyim.toml with a [workspace] section.
pub fn detect_workspace(start: &Path) -> Option<Workspace> {
    let mut current = if start.is_file() {
        start.parent()?.to_path_buf()
    } else {
        start.to_path_buf()
    };
    loop {
        let manifest_path = current.join("glyim.toml");
        if manifest_path.exists() {
            if let Ok(manifest) = load_manifest(&manifest_path) {
                if let Some(ws) = &manifest.workspace {
                    let members = resolve_member_globs(&current, &ws.members)?;
                    return Some(Workspace {
                        root: current,
                        members,
                        manifest,
                    });
                }
            }
        }
        if !current.pop() {
            return None;
        }
    }
}

/// Resolve glob patterns like "crates/*" to actual directories containing glyim.toml.
fn resolve_member_globs(
    root: &Path,
    patterns: &[String],
) -> Option<Vec<PathBuf>> {
    let mut members = Vec::new();
    for pattern in patterns {
        if pattern.ends_with("/*") {
            let dir = root.join(&pattern[..pattern.len() - 2]);
            if let Ok(entries) = std::fs::read_dir(&dir) {
                for entry in entries.flatten() {
                    if entry.path().is_dir() {
                        let member_toml = entry.path().join("glyim.toml");
                        if member_toml.exists() {
                            members.push(entry.path());
                        }
                    }
                }
            }
        } else {
            let member_path = root.join(pattern);
            if member_path.join("glyim.toml").exists() {
                members.push(member_path);
            }
        }
    }
    Some(members)
}
```

```rust
use std::fmt;
use std::path::PathBuf;

// ── Legacy type (backward compat) ────────────────────────────────────────

/// Simplified manifest with just name and version.
/// Used internally where full manifest fields aren't needed.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct SimpleManifest {
    pub name: String,
    pub version: String,
}

impl From<glyim_pkg::manifest::PackageManifest> for SimpleManifest {
    fn from(m: glyim_pkg::manifest::PackageManifest) -> Self {
        SimpleManifest {
            name: m.package.name,
            version: m.package.version,
        }
    }
}

impl fmt::Display for SimpleManifest {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result<(), fmt::Error> {
        write!(f, "{} v{}", self.name, self.version)
    }
}

/// Parse a glyim.toml string into a SimpleManifest.
pub fn parse_manifest(toml_str: &str) -> Result<SimpleManifest, ManifestError> {
    let full = glyim_pkg::manifest::parse_manifest(toml_str, "glyim.toml")?;
    Ok(SimpleManifest::from(full))
}

/// Load a SimpleManifest from a file path.
pub fn load_manifest(path: &Path) -> Result<SimpleManifest, ManifestError> {
    let content = std::fs::read_to_string(path)?;
    let file_name = path.to_string_lossy().to_string();
    parse_manifest(&content, &file_name)
}

/// Legacy error type for backward compatibility.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ManifestError {
    FileNotFound(PathBuf),
    Parse(String),
    MissingSection(&'static str),
    MissingField(&'static str),
}

impl fmt::Display for ManifestError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result<(), fmt::Error> {
        match self {
            Self::FileNotFound(p) => write!(f, "glyim.toml not found at {}", p.display()),
            Self::Parse(msg) => write!(f, "invalid TOML in glyim.toml: {msg}"),
            Self::MissingSection(s) => write!(f, "missing [{s}] section in glyim.toml"),
            Self::MissingField(f) => write!(f, "missing required field '{f}' in glyim.toml"),
        }
    }
}

impl std::error::Error for ManifestError {}
```
