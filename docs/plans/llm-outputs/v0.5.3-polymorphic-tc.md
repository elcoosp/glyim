# Polymorphic Type Checking with Instantiation Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable the type checker to correctly type-check generic implementation bodies (e.g., `impl<K, V> HashMap<K, V>`) by treating generic parameters (`K`, `V`) as valid, distinct inference variables rather than opaque errors.

**Architecture:** Implement *Instantiation* of generic types. When checking a generic body, map generic parameters (Symbols) to fresh TypeVars (Infer). When resolving field types on generic structs, substitute these TypeVars into the struct's definition, creating a concrete, albeit generic, type for the expression.

**Tech Stack:** Rust, `glyim-typeck` crate, `glyim-hir` crate.

## Chunk 2: Instantiation Logic in `check_fn`

Now we modify the type checker to map generic parameters (`K`) to inference variables (`?1`) when entering a generic function.

### Task 2: Map Generic Params to Fresh TypeVars

**Files:**
-   Modify: `crates/glyim-typeck/src/typeck.rs` (Function `check_fn`)

-   [ ] **Step 1: Modify `check_fn` to handle generic params**

We need to replace the current logic that inserts `HirType::Param`. We will instead create `Infer` vars.

Locate `fn check_fn(&mut self, f: &glyim_hir::HirFn)`.
Replace the section handling `f.type_params`.

```rust
        // ... inside check_fn ...
        
        // For generic functions, register type parameters as fresh inference variables.
        // This allows the body to be type checked "generically".
        let mut param_subst: std::collections::HashMap<Symbol, HirType> = std::collections::HashMap::new();
        
        for &tp in &f.type_params {
            let var = self.table.fresh_var(f.span);
            // We store the mapping of Symbol -> InferVar so we can use it later.
            // However, `self.env` usually expects HirType.
            // We will bind the Symbol to the Infer Var in the environment.
            // This makes "K" look like "?1" to the checker.
            
            let infer_ty = HirType::Infer(var);
            
            // We keep a local mapping for potential later use (though env lookup might suffice)
            param_subst.insert(tp, infer_ty);
            
            self.env.insert_global(tp, infer_ty, false);
        }
        
        // ... rest of check_fn continues ...
```

*Note:* We aren't using `param_subst` fully yet, but we've changed the environment binding.

```rust
        // we need to substitute the self type.
        // This is a bit manual for now. 
        // A heuristic: if params exist and the first param type is Generic matching the Impl target...
        
        // For this step, we just ensure the params are bound. 
        // The substitution on `self` happens naturally if we look up `K` in the env.
        // But `self` type is coming from HirItem which has `Param(K)`.
        // We need to substitute the param types of `self` using our `param_subst`.
        
        if let Some((_, self_ty)) = f.params.first() {
             if self_ty.has_param() {

                 let new_self_ty = crate::types::substitute_type(self_ty, &param_subst);

             }
        }
```

## Chunk 3: Structural Substitution in Field Access

Finally, we ensure that when we access `self.buckets`, where `self` is `HashMap<?1, ?2>`, we look up `buckets` in `HashMap<K,V>` (definition) and substitute `K->?1, V->?2`.

### Task 3: Substitute Field Types with Instantiated Args

**Files:**
-   Modify: `crates/glyim-typeck/src/typeck.rs` (Function `infer_field_access`)

-   [ ] **Step 1: Locate `infer_field_access`**

Find `fn infer_field_access(&mut self, obj: &HirExpr, field: Symbol, span: Span) -> HirType`.

-   [ ] **Step 2: Implement substitution logic**

Inside the match for `HirType::Generic(s, args)`:

```rust
        HirType::Generic(s, args) => {
            if let Some(ref idx) = self.hir_index
                && let Some(si) = idx.find_struct(*s)
                && let Some(&fi) = si.field_map.get(&field)
            {
                let field_ty_original = si.fields[fi].1.clone(); // e.g. Vec<Entry<K,V>>
                
                // If the struct is generic (has type params) and we have concrete args (the Infer vars),
                // we must substitute.
                if !si.type_params.is_empty() && !args.is_empty() {
                    // Create mapping: K -> ?1, V -> ?2
                    let sub: std::collections::HashMap<_, _> = si.type_params
                        .iter()
                        .zip(args.iter())
                        .map(|(&p, a)| (p, a.clone()))
                        .collect();
                    
                    // Apply substitution to field type
                    return crate::types::substitute_type(&field_ty_original, &sub);
                }
                
                return field_ty_original;
            }
            // ... error handling ...
        }
```
