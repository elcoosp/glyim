# v0.6.0 Implementation Plan (Revised for Actual Codebase)

> **Scope:** This plan covers the full macro system infrastructure for Glyim v0.6.0, built on the **real, existing codebase** (which already has a type checker, HIR, attribute parsing, and a basic `MacroContext` trait). The plan is divided into 7 phases, each adding incremental value while respecting the current compiler capabilities.

---

## Pre‑Requisites & Ground Truth

**Current codebase features (verified):**

- `glyim‑interner` → `Symbol`, `Interner`
- `glyim‑diag` → re‑exports `miette`, provides `Span` + converter
- `glyim‑lex` → full lexer including attribute tokens `#`, `[`, `]`
- `glyim‑parse` → AST with `Attribute`, `Item::FnDef` with `attrs: Vec<Attribute>`, structs, enums, impl blocks, generics, match, extern; `ExprKind::MacroCall` for `@name(expr)`
- `glyim‑hir` → full HIR including `HirItem` (Fn, Struct, Enum, Impl, Extern), monomorphization
- `glyim‑typeck` → complete type checker with struct/enum/impl/extern registration, expression/statement checking, generic type argument inference
- `glyim‑macro‑core` → already contains a minimal `MacroContext` trait (3 methods) and `expand::interpret_macro` (identity). Also has `Field` struct.
- `glyim‑cli` → uses `miette` for error reporting (`set_hook`, `MietteHandlerOpts`)

**Key insight:** The compiler can already parse and type‑check a significant subset of Glyim. Macro expansion is the main missing piece. We will therefore:
- Extend diagnostic reporting using existing `miette` infrastructure.
- Build upon the existing `MacroContext` trait and implement its methods against the type checker.
- Leverage the AST’s `Attribute` type to parse attribute arguments (only raw strings are available).
- Add lexer/parser support for `$` and `quote!` syntax, but keep macro invocation style consistent (`macro_name!` vs `@macro_name` – the parser already has `MacroCall`; we’ll handle `quote!` as a special macro).
- Provide a built‑in derive registry and composability checker.
- Add incremental expansion infrastructure and a debugger.

## Phase 1: Diagnostics, HirBuilder, Quote/Splice Parsing

### 1.1 Extend glyim-diag with macro‑error helpers

**Goal:** Provide a `DiagnosticBuilder` that adds multiple spans, notes, and expansion traces to a `miette::Diagnostic` without inventing a new type.

**File:** `crates/glyim-diag/src/builder.rs` (new)

**Tests:** 12 tests covering:
- Adding multiple labels (primary/secondary)
- Adding notes
- Adding expansion frames
- Converting to `miette::Report`
- Chaining methods

**Implementation:**

```rust
use miette::{Diagnostic, SourceSpan};
use std::fmt;

pub struct ExpansionFrame {
    pub macro_name: String,
    pub location: Option<crate::Span>,
}

pub struct DiagnosticBuilder {
    severity: miette::Severity,
    message: String,
    primary_span: Option<crate::Span>,
    labels: Vec<(crate::Span, Option<String>, bool)>,
    notes: Vec<String>,
    expansion_trace: Vec<ExpansionFrame>,
}
// impl methods: error(msg), warning(msg), note(msg), with_span(), with_label(), with_note(), with_expansion_frame(), build() -> miette::Report
```

Update `lib.rs` to re‑export `DiagnosticBuilder` and `ExpansionFrame`. Do **not** break existing `Diagnostic` re‑export from `miette`.

### 1.2 Create HirBuilder

**Goal:** Provide a fluent API to construct HIR nodes – the target for `quote!` expansion.

**File:** `crates/glyim-hir/src/hir_builder.rs` (new)

**Tests:** 22 tests covering all constructors: literals, binary/unary, block, if, while, for‑in, match, struct, enum, field access, call, method call, as, deref, let/assign statements.

**Implementation:** Reuses existing `HirExpr`, `HirStmt`, `HirBinOp`, `HirUnOp`. Builders like `HirBuilder::int_lit(42)` produce `HirExpr::IntLit { value:42, ... }`.

### 1.3 Lex and parse `$` token and `quote! { ... }` macro invocation

**Goal:** Recognize `$` in the lexer and parse `ident!` macro invocations (like `quote! { ... }`) as raw body text.

**Files:**
- `crates/glyim-syntax/src/kind.rs` – add `Dollar` variant
- `crates/glyim-lex/src/lexer.rs` – lex `$`
- `crates/glyim-parse/src/ast.rs` – add `ExprKind::Quote { body: String }`
- `crates/glyim-parse/src/parser/mod.rs` – handle `ident! { ... }` in atom parsing

**Tests:** 8 tests including tokenizing `$`, parsing `quote!`, nested braces, fallback to ident when no `!`.

**Implementation details:**
- Lexer: `'$' => SyntaxKind::Dollar`
- Parser: when encountering `ident` that is followed by `!` and `{`, consume all tokens inside braces (track depth), store raw body as `ExprKind::Quote { body }`.
- The existing `MacroCall` path is for `@name(...)`; we add a new branch for `name! { ... }`.

### 1.4 Lower `Quote` to a placeholder HIR expression

**File:** `crates/glyim-hir/src/lower.rs` – add arm for `ExprKind::Quote { body }` producing `HirExpr::IntLit(0)` (stub).

## Summary of Changes from Original Plan

| Issue | Original | Revised |
|-------|----------|---------|
| Diagnostic system | Custom `Diagnostic` struct | Uses `miette` + `DiagnosticBuilder` wrapper |
| MacroContext | New trait | Extended the existing minimal trait |
| Type‑checker integration | Deferred to v0.3.0 | Implemented immediately via `TypeckContext` |
| Lex/parse | Quote syntax as new token | Adds `$` token, leverages existing `MacroCall`‑like handling |
| HIR builder | Needed new module | Works with existing HIR types – purely additive |

This revised plan respects the actual state of the code, avoids breaking changes, and delivers a working macro infrastructure faster.
